# 埋点测试指南

## 🚀 快速开始

### 1. 访问测试页面
开发环境运行后，访问：`http://localhost:3000/tracking-test`

### 2. 测试步骤
1. 打开浏览器控制台（F12）
2. 点击测试按钮
3. 查看控制台日志和 Network 面板

---

## 📊 验证埋点数据

### 控制台日志
点击测试按钮后，控制台会显示：
```
[埋点上报] [
  {
    eventName: "test_event",
    page: "/tracking-test",
    action: "custom",
    properties: { test: "基础埋点测试" },
    timestamp: 1744567890123
  }
]
```

### Network 面板
1. 打开 Network 标签
2. 筛选 XHR/Fetch 请求
3. 查找发送到 `/api/tracking` 的请求
4. 查看 Request Payload 中的埋点数据

---

## ✅ 功能影响测试

### 测试方法
1. **正常功能测试**：使用所有页面功能
2. **埋点禁用测试**：点击"埋点已禁用"按钮，验证功能仍正常
3. **网络断开测试**：断开网络，验证功能不受影响

### 预期结果
- ✅ 埋点失败不影响业务功能
- ✅ 所有按钮和交互正常工作
- ✅ 页面性能无明显下降
- ✅ 控制台无红色错误

## 🛠️ 故障排查

### 问题1：看不到埋点日志
**解决**：确认开发环境（`import.meta.env.DEV === true`）

### 问题2：Network 没有请求
**原因**：后端接口未实现（这是正常的）
**验证**：控制台日志显示埋点已触发即可

### 问题3：功能受影响
**解决**：检查是否有其他错误，埋点代码有错误保护
