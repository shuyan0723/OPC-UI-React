# OPC UI 需求文档

## 1. 导航与全局交互

### 1.1 导航栏锚点保持
- **需求**：点击左侧导航栏的某一模块时，左侧导航栏应保持在当前滚动位置
- **约束**：严禁自动跳转或重置到导航栏的最上方

### 1.2 AI悬浮按钮
- **需求**：实现一个全局通用并且可移动的AI悬浮按钮
- **要求**：确保在应用的任何页面、任何路由下均可点击唤起并使用

### 1.3 响应式布局
- **需求**：页面布局需具备自适应能力
- **要求**：根据屏幕实际大小自动调整至最佳显示状态

---

## 2. "个性化设置"菜单重构

### 2.1 移除独立导航
- **需求**：从主导航栏中彻底移除以下三个独立的入口
  - "行业定制"
  - "角色定制"
  - "阶段定制"

### 2.2 新增悬停交互
- **需求**：在"个性化设置"菜单项上增加鼠标悬停（Hover）事件
- **交互逻辑**：当光标悬停在"个性化设置"上时，在其右侧展开显示三个子选项：
  - 行业定制
  - 角色定制
  - 阶段定制

### 2.3 状态保持逻辑
- **需求**：只有当用户点击上述三个子选项之一时，右侧主内容区域才切换显示相应界面
- **约束**：若未点击（仅悬停或鼠标移开），右侧界面必须保持在上一操作所在的界面，不发生跳转

---

## 3. "企业发展阶段"逻辑字段更新

### 3.1 阶段判定规则
根据"企业成立时间"自动匹配阶段描述：

| 成立时间 | 阶段 | 描述 |
|---------|------|------|
| 0年 < 成立时间 ≤ 3年 | 初创期 | 产品验证，市场探索 |
| 3年 < 成立时间 ≤ 7年 | 成长期 | 规模扩张，模式优化 |
| 成立时间 > 7年 | 成熟期 | 稳定运营，持续创新 |

### 3.2 自动化判定
- **禁用手动选择**："初创期"、"成长期"、"成熟期"这三个选项不可手动点击选择
- **联动逻辑**：仅当用户输入或选择完"企业成立时间"后，系统根据上述规则自动判定并显示企业所在的阶段

### 3.3 功能新增
- **需求**：在"企业成立时间"字段右侧增加一个"修改"按钮，用于编辑成立时间

## 附录：类型导入问题修复

### 问题描述
TypeScript 无法识别 `import type { Task } from '@types'` 路径。

### 原因分析
Vite 配置与 TypeScript 配置不一致：

| 配置文件 | `@types` 别名 |
|---------|--------------|
| vite.config.js | `'@types': path.resolve(__dirname, './src/types')` |
| tsconfig.json | `"@types/*": ["./src/types/*"]` |

### 推荐解决方案
在 tsconfig.json 的 paths 中添加：
```json
"@types": ["./src/types"]
```
